# PULA Biz App Scheme

## Current testing path

PULA Biz now ships as an installable Progressive Web App. The same deployed website becomes:

- Web app in any browser.
- Android app through Chrome install.
- iPhone app through Safari Add to Home Screen.
- iPad app through Safari Add to Home Screen.
- Windows desktop app through Chrome or Edge install.
- Mac desktop app through Chrome, Edge, or Safari Add to Dock.

All versions use the same React/Firebase build. When GitHub deployment publishes a new web build, installed apps receive the update from the service worker automatically the next time the app is opened or while it is running.

## User install flow

Website visitors can use the Apps section on the homepage:

- Android App: opens the native browser install prompt where supported.
- iPhone App: sends the user to the install URL and shows Safari instructions.
- iPad App: sends the user to the install URL and shows iPad Safari instructions.
- Windows App: opens the desktop browser install prompt where supported.
- Mac App: opens the desktop browser install prompt where supported, or shows Add to Dock guidance.
- Download APK: links to `/downloads/pula-biz-latest.apk`.
- Download Mac DMG: links to `/downloads/pula-biz-mac-latest.dmg`.

The Mac DMG is currently generated in `public/downloads/pula-biz-mac-latest.dmg`. The APK link is direct, but the APK file must be generated with an Android build toolchain and placed at `public/downloads/pula-biz-latest.apk`.

Use this URL for testing on devices:

```text
https://pulabiz.com/#apps
```

## GitHub to all platforms

Use one release path:

1. Push code to GitHub.
2. Existing deployment builds and publishes the web app.
3. The browser app, Android installed app, iPhone installed app, iPad installed app, Windows installed app, and Mac installed app all load the same new deployment.
4. No manual app update is required for the testing-stage installed apps.

## Native package path

For direct package downloads, build and upload signed files into `public/downloads`:

- Android direct package: `public/downloads/pula-biz-latest.apk`.
- Mac direct package: `public/downloads/pula-biz-mac-latest.dmg`.
- Future Windows package: `public/downloads/pula-biz-windows-latest.exe` or `public/downloads/pula-biz-windows-latest.msi`.

These files should be generated by a native wrapper such as Capacitor, Tauri, or Electron. Most product changes should still remain in the web app so installed apps continue receiving over-the-air updates.

## Later store path

When the product is ready for App Store and Play Store, wrap this same PWA with a native shell such as Capacitor:

- Android: generate an APK/AAB for Play Store or direct internal testing.
- iOS/iPadOS: generate an iOS app for TestFlight/App Store.
- Windows/macOS: generate signed desktop installers with Tauri or Electron if store-style desktop distribution is needed.
- Keep most changes in the web app so normal product updates still ship over the air.
- Use native releases only when changing app permissions, native plugins, splash screens, store metadata, or native shell configuration.

This keeps the current testing flow fast while leaving a clean migration path to official stores and signed desktop installers.
